paper

Fast and effective optic disk localization based on convolutional neural network
笔记见纸质版。pdf在这里：


预测的是od中心，只要与GT的欧式距离小于od半径的一半，都算正确。

  1. 在训练集中，提取固定大小（例如56*28）的局部图像（patch）。为保证数据均衡，一半为od patch(patch的中心点距离od真实中心欧式距离小于某值)，一半为非od patch。
  2. 训练一个二分类的CNN，可以预测输入的patch为视盘的概率(0-1之间)。
  3. 用训练好的CNN，将一张眼底图，转化为一张相同大小的概率图，其中（x,y）的值表示以（x,y）为中心的patch是od的概率。
  4. 对概率图加权投票，即用高斯滤波处理。概率最大的位置预测为od中心。
  • 亮点：RGB -> RGV; 输入矩形而不是正方形，有更多血管信息。

Search in Rotated Sorted Array

Input: nums = [4,5,6,7,0,1,2], target = 0
Output: 4

  • 二分搜索法的关键在于获得了中间数后，判断下面要搜索左半段还是右半段
  • 以mid为界，至少有一边的数是有序的：
    • 如果mid<right，则右边是有序的
    • 如果left<mid，则左边是有序的

一个博客中的思路

Jump Game

题目
贪心，98%。从数组的第一个数开始，每次可以走nums[i]步，问是否可以走到最后一个数。
Example 1:
Input: [3,2,1,0,4]
Output: false
Explanation: You will always arrive at index 3 no matter what. Its maximum jump length is 0, which makes it impossible to reach the last index.

思路
用step表示当前最多可以走几步，每走一步就减一，如果nums[i]大于step就更新step。

Two Pointer Approach
非常巧妙。用两个指针，一开始一个在头一个在尾。每次把短的木板向内移动。原因如下：

If we try to move the pointer at the longer line inwards, we won’t gain any increase in area, since it is limited by the shorter line.
But moving the shorter line’s pointer could turn out to be beneficial.
如果移动较长的木板，收到短木板的影响，height不变而width减小，面积只会减小。

Subset

可以一位一位的往上叠加，如[1,2,3]：

  1. 最开始是空集。
  2. 处理1，在空集上加1，为[1]。
  3. 处理2，在之前的子集（[]、[1]）基础上，加2，得到[2]，[1, 2]
  4. 处理3，可得[3], [1, 3], [2, 3], [1, 2, 3]
  • 对有n个元素的数组处理n次。每次在前i-1次处理的基础上，对每个集合加上nums[i]。每次处理后，subset数量*2。
  • 这种叠加的思路，有点像这道dpBitwise-ORs-of-Subarrays
