Linked List Cycle

快慢指针法
即用两个指针从head出发，fast每次走2步，slow每次走一步。
快慢指针法可以处理很多链表成环的问题:

  • 判断是否成环
  • 求环长度
  • 求环的入口
  • 求带环链表的总长度

  • 判断是否成环：如果成环，快慢指针必然相遇，且相遇时fast刚好比slow多走一个环（可以用数学归纳法来证明）。如果fast在移动过程中指向NULL，则不成环
  • 求环的长度：从出发到相遇的时间t
  • 求环的入口：从相遇点用2个指针，每次走一步，这2个指针一定会在环的入口相遇
  • 求带环链表的总长度：在找到环入口后，用head到入口的长度+环的长度

Find the Duplicate Number

放上ju头的题解leetcode 287
因为数组的范围是1-n，可以把数字之间的转移关系变成一个有向图。nums[index]->nums[nums[index]]
[3,1,3,4,2]:

  • 数字3指向index=3的数字4
  • 数字4指向index=4的数字2
  • 数字2指向index=2的数字3
  • 数字1指向index=1的数字1
    3->4->2->…
    nums[x]=val,nums[y]=val：nums[x]->nums[val]、nums[y]->nums[val]，即出现2次val->nums[val]，成环。

从这里看出来3，4，2形成了一个环，而环的入口点是3，这个问题就转换成了找成环链表的环入口。
看上图中，1是单独出来的，当数组中index位置的值也为index时，会出现单独成环的情况。
但是如果从index=0的地方开始，因为数组取值为1-n，不会单独成环。

Majority Element

摩尔投票法：

  • 空间复杂度O（1），时间复杂度O（n）。
  • 每次从序列里选择两个不相同的数字抵消，最后剩下一个数字或几个相同的数字，就是出现次数大于n/2的那个。
  • 必须保证该数存在

Majority Element II

显然，出现次数大于1/3的数字最多有2个，也可能一个没有
参考思路
三三抵消，最后会剩下两个candidates，但是注意此时不是谁占多数谁是最终结果，反例[1, 2, 3, 1, 2, 3, 1, 2, 3, 1, 4, 4] 三三抵消后剩下 [1, 4，4] 4数量占优，但结果应该是1，所以三三抵消后，再遍历一遍找1和4谁数量超过了len(nums)/3。

Reverse Linked List

  • p->pr->q
  • 用q=pr->next保留pr的后继结点，用pr->p使相邻结点反转
  • 将指针向后移动: p=pr，pr=q
  • 一开始，让p=NULL，pr=head(第一个结点)
  • 在pr/q指向NULL时，结束

Rotate Image

旋转180度
方法一：矩形，开辟新空间

  • 开辟了新空间
  • 矩形都适用
  • matrix2[i][j]=原矩阵中与之中心对称的元素
  • 旋转后长宽不变

方法二：正方形，原地交换
也可以在原地swap：将刚好一半的元素，与中心对称处的元素交换。但是要注意，对角线下方（不算正中间）的元素不要被二次交换：
123 会 987
456 变 654
789 成 321

  • 1<->9、2<->8、3<->7
  • 4<->6、5<->5
  • 如果7再与3换，就换回去了。所以将对角线也等分，保证对角线上的元素只交换一次。
    注意这种方法对于正方形（n*n）才管用。因为只有正方形，才可以沿对角线等分。 旋转90度

Permutations

不快，16%。想法很简单，每次插入一个数字，比如[1,2,3,4]，依次插入1、2、3、4:

  • 插入1:[1]
  • 插入2:[2,1],[1,2]
  • 插入3:[3,2,1],[2,3,1],[2,1,3];[3,1,2],[1,3,2],[1,2,3]
  • 插入4…
    每插入一个新数，遍历上一次插入的结果，在每个结果的0-size位置上插入新数。
